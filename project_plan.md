# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제: 하이랄 성 잠입 미니게임(젤다의 전설 신들의 트라이포스 바탕)

---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

내 설명: 과거 닌텐도로 즐기던 명작 게임 '젤다의 전설: 신들의 트라이포스'의 하이랄 성 탈출 파트를 파이썬 콘솔 환경에서 직접 플레이할 수 있도록 재현하고자 합니다. 2차원 리스트와 함수, 조건문/반반문을 활용해 플레이어가 성을 탐험하고 경비병과 싸우며 젤다 공주를 구출하는 성취감을 느낄 수 있는 미니게임을 구현하는 것이 목표입니다.
---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 사용자의 이동 명령: 플레이어가 가고 싶은 방향 (W: 위, A: 왼쪽, S: 아래, D: 오른쪽)
- 전투 행동 선택: 경비병을 만났을 때 선택하는 행동 (예: 1번 공격, 2번 도망 등)

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 위치 업데이트 및 벽 충돌 검사: 플레이어의 입력에 따라 2차원 리스트로 구성된 하이랄 성 지도 안에서 링크의 행과 열 위치를 변경하되, 벽('W')이 있으면 이동하지 못하게 막음
- 이벤트 판정: 이동한 칸에 경비병('E')이나 보물상자('T')가 있을 때, 각각 체력을 깎거나(또는 전투를 치르거나) 아이템을 획득하도록 처리함
- 상태 업데이트: 전투 시 링크의 체력과 몬스터의 체력(2차원 리스트 내 데이터)을 계산하여 실시간으로 반영함

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 현재 상황 시각화: 현재 하이랄 성의 지도 모양과 링크('P'), 경비병('E') 등의 위치를 텍스트로 화면에 출력
- 링크의 상태 표시: 현재 링크의 남은 체력(하트 수)과 보유 아이템 출력
- 게임 결과 메시지: 젤다 공주를 구출했을 때의 축하 메시지 또는 체력이 0이 되어 게임 오버되었을 때의 안내 메시지 출력

## 4. 2차원 리스트 설계

내 프로그램에서 사용할 2차원 리스트의 이름:monsters = [
    ["초록 경비병", 20, 5, "일반"],
    ["빨간 경비병", 40, 10, "정예"],
    ["철퇴 기사", 80, 20, "보스"]
]
### 한 행의 의미
하이랄 성 던전 안에 존재하는 '개별 몬스터(적) 1마리의 모든 정보'를 의미합니다.

### 각 열의 의미
| 열 번호 | 의미 | 예시 |
|---:|---|---|
| 0번 열 |몬스터의 이름 (문자열)  | 초록 경비병 |
| 1번 열 | 몬스터의 현재 체력 (정수형) | 20 |
| 2번 열 | 몬스터의 공격력 (정수형) | 5 |
| 3번 열 | 몬스터의 등급 또는 특징 (문자열) | 일반 |

## 5. 함수 설계

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
| display_status| 현재 링크의 체력과 하이랄 성 지도 상황을 화면에 보여줌 | 성 지도 리스트, 링크의 체력 | 화면에 텍스트 맵과 체력 출력 |
| move_player | 사용자 입력(W,A,S,D)에 따라 플레이어의 위치 좌표를 안전하게 변경함 | 현재 위치, 입력된 방향 문자 | 변경된 새로운 위치 좌표 |
| fight_monster |경비병을 만났을 때 monsters 리스트의 데이터를 바탕으로 턴제 전투를 진행함  | 만난 몬스터의 인덱스 번호 | 전투 결과 (승리/패배/도망) |

## 6. 사용할 제어구조

### 조건문을 사용할 부분
내 계획:이동 제한 조건: 사용자가 이동하려는 칸이 벽("W")인 경우 이동을 취소하고 경고를 출력할 때 사용합니다.

이벤트 발생 조건: 링크가 도달한 칸의 글자가 "E"면 전투를 실행하고, "T"면 아이템을 획득하며, "Z"면 게임 승리로 처리할 때 활용합니다.

전투 승패 판정: 링크나 몬스터의 체력이 0 이하가 되었는지 검사할 때 사용합니다.


### 반복문을 사용할 부분

내 계획:게임 메인 루프: 링크가 죽거나(체력 <= 0) 젤다 공주를 구할 때까지(승리 조건 만족) 전체 게임 과정을 계속해서 반복 실행합니다.

지도 출력: 2차원 리스트 형태인 성 지도의 모든 행과 열을 하나씩 꺼내어 화면에 바둑판 모양으로 출력할 때 중첩 반복문(for 루프)을 사용합니다.

## 7. 예상되는 예외 상황

1. 잘못된 키 입력: 사용자가 방향키(W, A, S, D)나 전투 메뉴(1, 2)가 아닌 엉뚱한 문자나 숫자를 입력했을 때 프로그램이 멈추지 않고 "잘못된 입력입니다."라고 안내한 뒤 다시 입력받도록 예외 처리가 필요합니다.
2. 지도 경계 이탈: 플레이어가 맵의 가장자리에서 바깥쪽으로 나가려고 할 때 리스트 인덱스 범위 오류(IndexError)가 나지 않도록 경계를 벗어나는 이동을 막아야 합니다.
